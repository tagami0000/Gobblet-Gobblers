# 要件定義書

## プロジェクト概要

| 項目 | 内容 |
|---|---|
| プロジェクト名 | Gobblet Gobblers |
| 種別 | ブラウザゲーム（個人開発・ポートフォリオ） |
| 目的 | 技術学習および成果物としてのポートフォリオ |
| プラットフォーム | Web ブラウザ（PC 専用） |

## 対戦形式

| 項目 | 内容 |
|---|---|
| 対戦形式 | ローカル 2 人対戦（同一画面でターン交代） |
| プレイヤー名 | 固定（Player 1 / Player 2） |
| AI 対戦 | 対象外（将来拡張候補） |
| オンライン対戦 | 対象外 |

## 機能一覧

### MVP（今回実装対象）

| 機能 | 説明 |
|---|---|
| ゲームボード表示 | 3×3 グリッドの描画 |
| コマ手持ちエリア | 大・中・小 各 2 個の残数表示、使用済みはゴースト表示 |
| コマ選択・配置操作 | クリック 2 ステップ（選択 → 配置先クリック） |
| 配置バリデーション | 置けないマスへの操作を無効化 |
| 勝敗判定 | 3 並び検出、勝者表示 |
| リセット機能 | ゲームの再スタート |
| 手番表示 | 現在のプレイヤーを常時表示 |

### スコープ外

| 機能 | 理由 |
|---|---|
| AI 対戦 | 実装コストが高い。将来拡張候補 |
| オンライン対戦 | バックエンド・WebSocket が必要 |
| Gobblet（4×4 版） | 基本版完成後に検討 |
| プレイヤー名カスタマイズ | MVP には不要 |
| モバイル対応 | PC 専用で十分 |

## 設計上の判断

| 判断事項 | 決定内容 | 理由 |
|---|---|---|
| 操作方式 | クリック 2 ステップ（選択 → 配置先） | ドラッグ&ドロップより実装がシンプル |
| 覆われたコマの表示 | 表示しない | 原作ルール準拠。戦略的な駆け引きが生まれる |
| UI スタイル | リッチ・インタラクティブ | ポートフォリオとして視覚的な訴求力を重視 |
| 特殊ルール採用 | ボード移動で相手が 3 並び → 相手の勝ち | 原作ルール準拠 |

## 技術スタック

| 用途 | 技術 | 選定理由 |
|---|---|---|
| フレームワーク | React 18 | コンポーネント分割と状態管理が扱いやすい |
| 言語 | TypeScript | 型安全性によりゲームロジックのバグを防ぎやすい |
| ビルドツール | Vite | 高速な開発サーバー |
| スタイリング | Tailwind CSS v4 | ポートフォリオとして見栄えが出やすい |
| 状態管理 | useReducer（React 組み込み） | 外部ライブラリ不要の規模感 |
